## Архитектура
В качестве архитектуры приложения используется MVVM + Clean Architecture. Такой подход был выбран по следующим причинам:
- каждый слой отвечает за свои задачи
- код становится проще расширять, сопровождать и тестировать
- подмена источников данных не затрагивает бизнес-логику и др.

___
## Решения
- Состояние UI. Для отображения загрузки, получения данных и ошибок используются sealed class и StateFlow во ViewModel
- Внедренение зависимости. В качесте DI используется Hilt. Он отлично подходит для небольших и средних проектов.
- Логирование сетевых запросов. Для отслеживания сетевых запросов и ответов был добавлен HttpLoggingInterceptor.
- Передача данных между фрагментами. Для передачи данных между фрагментами используется navArgs(), потому что он типобезопасен и позволяет передавать сложные структры данных.
- Оптимизация списка. Для оптимизации списка используется DiffUtil, который позволяет не перерисовывать весь список, а только те данные, которые поменялись.

___
## Используемые технологии
- ЯП: Kotlin
- UI: XML
- Сеть: Retrofit2 + OkHttp
- DI: Hilt
- Архитектура: MVVM + Clean Architecture
- Асинхронные вызовы: Kotlin Coroutines
- Потоки данных: Kotlin Flow
- Навигация: Navigation Architecture Component
- Media: ExoPlayer
